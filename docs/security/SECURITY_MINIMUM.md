# Security Minimum

## Минимальные правила PRD-0

- HTTPS only.
- OpenWebUI не публикуется напрямую наружу.
- Публичная регистрация отключена или пользователи попадают в `pending`.
- Один администратор, 3-4 обычных пользователя.
- API-ключи только на сервере.
- Реальный `.env` не коммитится.
- Backup с `.env` хранится только server-local.

## Пользовательские ограничения

Пользователям запрещено отправлять в чат:

- пароли;
- токены;
- API-ключи;
- приватные SSH-ключи;
- OAuth secrets;
- коммерчески чувствительные документы без отдельного разрешения.

## Админские действия

Администратор должен:

- создать пользователей вручную;
- проверить, что обычный пользователь не видит API-ключ;
- отключить signup после создания пользователей;
- проверить backup;
- собрать обратную связь после 1-2 недель.

## Не делаем в PRD-0

Не внедряем SSO/OIDC, сложный RBAC, DLP, корпоративный аудит и SIEM. Это future scope после подтверждения потребности.
